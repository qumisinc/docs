## Features

### Onboarding v2

<Tip>
  New users will now see a guided walkthrough when they first use Qumis, helping them understand key features and generate their first report successfully.
</Tip>

- Step-by-step guidance through the platform
- Progress saves automatically — users can resume anytime
- Celebration moment when they complete their first report

**Linear:** [ENG-1465](https://linear.app/qumis/issue/ENG-1465)

### Report Progress Tracking

<Tip>
  Users can now see exactly what's happening while their report generates, with real-time status updates and progress indicators.
</Tip>

- Live progress bar during report generation
- Shows current processing phase (e.g., "Analyzing documents", "Generating insights")
- Clearer field hierarchy in checklist reports

**Linear:** [ENG-1517](https://linear.app/qumis/issue/ENG-1517), [ENG-1518](https://linear.app/qumis/issue/ENG-1518)

### Chameleon Analytics

<Tip>
  We've integrated Chameleon to better understand user behavior and improve the product experience.
</Tip>

- Understand how users navigate the platform
- Identify friction points in workflows
- Controlled via feature flags — can be enabled/disabled as needed

**Linear:** [ENG-1387](https://linear.app/qumis/issue/ENG-1387)

### Improved Checklist Detection

<Tip>
  Enhanced detection of nested lists and repeatable sections in insurance checklists for more accurate data extraction.
</Tip>

- Better handling of complex insurance forms
- Accurate detection of numbered/nested items
- Smarter extraction of repeating sections

**Linear:** [ENG-1484](https://linear.app/qumis/issue/ENG-1484)

### Feature Flag System

<Tip>
  New infrastructure for conditional UI rendering, enabling controlled rollouts and A/B testing.
</Tip>

- Gradual rollout of new features to specific users
- A/B testing capabilities
- Quick enable/disable of features without deployment

**Linear:** [ENG-1574](https://linear.app/qumis/issue/ENG-1574)

### Credits System Updates

<Tip>
  Renamed "subscription reports" to "credits" with improved tracking and display throughout the app.
</Tip>

- "Reports" renamed to "Credits" throughout the app
- New trial end indicator for users approaching limits
- Updated pricing display page

**Linear:** [ENG-1502](https://linear.app/qumis/issue/ENG-1502)

## Improvements

- **Help Center link** now points to our documentation site ([ENG-1571](https://linear.app/qumis/issue/ENG-1571))
- **Docs authentication** via Mintlify OAuth for docs.qumis.ai ([ENG-1531](https://linear.app/qumis/issue/ENG-1531))
- **Configurable credits** — credit charging can be toggled via environment variable
- **Performance tracking** — new workflow benchmarking for monitoring report generation times
- **Better logging** — warnings for large callback requests (>6MB)
- **Error handling** — improved handling of JSON serialization errors
