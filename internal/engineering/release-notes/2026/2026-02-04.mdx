## Summary

Major release including onboarding v2, real-time workflow progress tracking during report generation, Chameleon analytics integration, and improved checklist schema detection for nested lists.

## Features

<AccordionGroup>
  <Accordion title="Onboarding v2" icon="sparkles">
    New user onboarding flow with step tracking, first report celebration, and resume capability.

    **Linear:** [ENG-1465](https://linear.app/qumis/issue/ENG-1465)

    **Changes:**
    - New onboarding controller and endpoints
    - Step state persistence in database
    - First report completion tracking
    - Confetti celebration on first report
  </Accordion>

  <Accordion title="Workflow Progress Tracking" icon="sparkles">
    Real-time progress display during report generation showing current step and phase.

    **Linear:** [ENG-1517](https://linear.app/qumis/issue/ENG-1517), [ENG-1518](https://linear.app/qumis/issue/ENG-1518)

    **Changes:**
    - Workflow steps table and API endpoints
    - Progress component with loading animation
    - Phase tracking and deduplication
    - Field name hierarchy display in checklist viewer
  </Accordion>

  <Accordion title="Chameleon Analytics Integration" icon="sparkles">
    User analytics integration for product insights and user behavior tracking.

    **Linear:** [ENG-1387](https://linear.app/qumis/issue/ENG-1387)

    **Changes:**
    - Backend integration with cached user stats
    - Frontend event tracking
    - Feature flag controlled rollout
  </Accordion>

  <Accordion title="Improved Template Schema Detection" icon="sparkles">
    Enhanced detection of nested lists and repeatable sections in insurance checklists.

    **Linear:** [ENG-1484](https://linear.app/qumis/issue/ENG-1484)

    **Changes:**
    - Heuristic-based schema detection for document-driven elements
    - Recursive schema for nested checklist elements
    - List instance handling with hierarchy path tracking
    - Document distillation with batch processing
  </Accordion>

  <Accordion title="Beta Feature Flag System" icon="sparkles">
    Conditional UI rendering based on feature flags for controlled rollouts.

    **Linear:** [ENG-1574](https://linear.app/qumis/issue/ENG-1574)

    **Changes:**
    - Beta feature flag infrastructure
    - Billing and signup flow toggles
    - Credit management feature flag
  </Accordion>

  <Accordion title="Credits System Refactoring" icon="sparkles">
    Renamed subscription reports to credits with configurable usage.

    **Linear:** [ENG-1502](https://linear.app/qumis/issue/ENG-1502)

    **Changes:**
    - CreditCost model for report and chat costs
    - Subscription credits across the application
    - Trial end indicator component
    - Price list page
  </Accordion>
</AccordionGroup>
